# Toloka — верификация событий на reasoning-трассах

Статический вьюер (GitHub Pages) для того, чтобы читать reasoning-трассу целиком и верифицировать
события, которые на ней нашли автоматические разметчики («агенты»): подтвердить, отклонить как
ложное или переразметить тип. По итогам считается score каждого агента.

Агенты: `regex` (детектор-регулярки), `Claude`, `DeepSeek`, `Qwen`.

Живёт на https://karpovilia.github.io/toloka/

## Как это работает

1. Слева — список трасс (584). Открываешь трассу — она показана целиком, слева от текста «карта
   reasoning» (спаны + интенсивность λ по Хоксу + метки событий).
2. На каждом событии агентов — инлайн-контрол: ✓ подтвердить (реально это событие данного типа),
   ✗ отклонить (ложное срабатывание), либо выбрать другой тип. Событие = кластер срабатываний
   одного типа в окне ±1 сегмента; в скобках видно, какие агенты его нашли.
3. Выбираешь строку (клик по строке; клавиши `j`/`k` = перейти к след./пред. событию). Справа —
   инфа о ВЫБРАННОЙ строке: оператор спана (+что кодирует), λ, текст, ссылка на строку, и события
   на этой строке с кнопками ✓/✗/тип. Вердикт с клавиатуры — `1` да / `2` нет (первому событию
   строки); между трассами `[` / `]`. Кнопка «перейти к след. событию ▶» — в правой панели.
4. Сохраняешь результат: «экспорт» (файл) или «💾 GitHub» (коммит из браузера).
5. `python3 build/score_agents.py` считает score агентов.

Разметка живёт в localStorage браузера (ключ = candId `<trace_file>|s<seg>|<type>`).

## Реальность данных

Срез трассы = множество LLM-агентов, реально её разметивших (C = Claude, D = DeepSeek,
Q = Qwen; regex пересчитывается детектором на каждой трассе). После полного Qwen-прогона
корпуса (2026-07) появился настоящий 4-way: срез CDQ (regex+Claude+DeepSeek+Qwen) — 1430
трасс корпуса; остальные комбинации — CD, DQ, Q, D. Во вьюер уходит топ конфликтных сайтов
(cap 6000): пол в 300 сайтов каждому срезу, остаток — пропорционально числу LLM-vs-LLM
конфликтов (prio>=3), отбор внутри среза — round-robin по ячейкам модель×бенчмарк (иначе
кап срезал бы хвост алфавита: musique, qwen-модели). Покрыты все 5 бенчмарков (math500,
gpqa_diamond, hle, hotpotqa, musique) и все 3 модели. Точные счётчики сборки — в
`data/build_summary.json`.

## Карта reasoning

Слева от текста — вертикальный гуттер на каждый сегмент (синхронный скролл):
- полоса оператора — операторный спан сегмента (SETUP/DERIVING/EXPLORING/VERIFYING/REVISING/
  CONCLUDING/GROUNDING) из dual-разметки (DeepSeek, если размечал трассу, иначе Qwen);
- красная полоса λ — измеренная интенсивность событий по Хоксу (ширина/яркость ∝ λ);
- глифы — события: ◇ branch, ◄ backtrack, ✗ failed_attempt, • прочие типы;
- клик по карте — попап с распределением λ по типам событий в этой точке (drill-down).

Интенсивность: `build/build_trace_maps.py` фитит пер-типовый self-exciting Hawkes (μ,α,β на тип,
ось = seg_id) на union-событиях корпуса; λ(s) = Σ_t λ_t(s). Параметры — в `data/trace_maps_meta.json`.
Быстрый пересчёт только λ — `--refit`. Данные карты — в `data/traces/*.json` (events/spans/lam).

## Скоринг агентов

`build/score_agents.py` по `annotations/annot_*.json`:
- gold по кандидату = мажоритарный вердикт разметчиков;
- detection precision агента = доля подтверждённых (✓ или переразмечен) среди размеченных
  кандидатов, где агент участвовал (± Wilson CI);
- type accuracy = доля верного типа (✓) среди реальных событий агента;
- межаннотаторное согласие: Cohen κ (2) / Fleiss κ (≥3).
- recall НЕ измеряется — пропущенные события в этой модели не размечаются (только события агентов).

## Сохранение результата

- Файлом: «экспорт» -> `annot_<ник>.json` -> в `annotations/` -> `git push`.
- Коммитом из браузера («💾 GitHub»): owner/repo/branch/path + fine-grained PAT (`contents:write`);
  файл `<path>/annot_<ник>.json` уходит в api.github.com. Работает с приватным репо. Токен — только
  в localStorage браузера, в страницу не зашит.

## Deep-link

`#trace=<trace_file>&seg=<N>` открывает конкретную трассу на событии; кнопка «🔗 ссылка» копирует URL.

## Сборка данных

    python3 build/build_conflicts.py --cap 6000 --radius 12   # (легаси, задаёт набор 584 трасс)
    python3 build/build_trace_maps.py                          # события/спаны/λ в data/traces/*.json
    python3 build/build_traces_index.py                        # data/traces_index.json (список трасс)

## Структура

    index.html, styles.css, app.js   вьюер v2 (чистая статика)
    config.json                      пути к данным
    data/traces_index.json           список трасс (список слева)
    data/traces/<cell>__<qid>.json   трасса: segments + events + spans + lam
    data/trace_maps_meta.json        параметры Хокса по типам

    annotations/                     сюда кладут annot_<ник>.json
    build/                           build_trace_maps.py · build_traces_index.py · score_agents.py · test_viewer_smoke.js
